# Java POO Exceptions – Metrics Analyzer

Proyecto desarrollado en Java que simula el cálculo de métricas en un contexto backend, con el objetivo de practicar Programación Orientada a Objetos y el manejo correcto de excepciones personalizadas.

## Objetivo del proyecto
Aplicar buenas prácticas de manejo de errores en Java mediante el uso de excepciones checked y unchecked, estructurando la aplicación en capas para mejorar la legibilidad y el mantenimiento del código.

## Arquitectura del proyecto
La aplicación está organizada por paquetes siguiendo una separación lógica de responsabilidades:

- app: punto de entrada de la aplicación
- service: lógica de negocio y validaciones
- model: entidades del dominio
- exception: excepciones personalizadas

## Conceptos aplicados
- Programación Orientada a Objetos (POO)
- Manejo de excepciones personalizadas
- Excepciones checked y unchecked
- Separación de responsabilidades
- Organización por capas

## Funcionalidades
- Cálculo de métricas a partir de datos de entrada
- Validación de datos del usuario
- Manejo controlado de errores y excepciones
- Flujo de ejecución claro y predecible

## Tecnologías utilizadas
- Java
- Programación Orientada a Objetos
- Manejo de Excepciones

## Ejecución
1. Clonar el repositorio
2. Abrir el proyecto en IntelliJ IDEA o Eclipse
3. Ejecutar la clase AppMetricas

## Contexto
Este proyecto forma parte del proceso de aprendizaje de Java backend y está orientado a fortalecer la gestión de errores y la organización del código, habilidades fundamentales para el desarrollo de aplicaciones backend más complejas.
